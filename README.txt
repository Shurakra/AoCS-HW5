5. Задача о каннибалах. Племя из n дикарей ест вместе из большого горшка,
который вмещает m кусков тушеного миссионера. Когда дикарь хочет
обедать, он ест из горшка один кусок, если только горшок не пуст, иначе
дикарь будит повара и ждет, пока тот не наполнит горшок. Повар, сварив
обед, засыпает. Создать многопоточное приложение, моделирующее обед
дикарей. При решении задачи пользоваться семафорами.

Иcпользовал модель "Производители и потребители"/"Клиенты и серверы"(скорее второе тк повар "спить" ждет пока ему не сообщает "запрос" на новых вкусных мессионеров). Точнее "WriterReaders". 

Модель: Некоторым объектом могут пользоваться много "читателей", как только все "читатели" заканчивают рабоать с объектом его сможет начать изменять "писатель". 

Ввод: при вызове исполняемого файла необходимо указать два аргумента n m(n - популяция каннибалов 1 - 100, m - объем котла каннибалов 1 - 1000).

1. Java multithreading readerwriter
2. Reader Writer Locks Java
3. Reader Writer Locks C#
